Si erreur, par exemple si j'ai entretemps rajouté un ReadMe : The remote repo was initialized/updated separately (e.g., README added on GitHub)

Pull the remote changes first

git pull

Attention il apparait un message : Astuce (éviter ce message à l’avenir)
git pull --rebase

If you want a cleaner history (rebase instead of merge)
git pull --rebase

Puis refaire :
git push

If you’re sure you want to overwrite the remote (⚠️ dangerous)
git push --force


Pour annuler les commit
git resert --hard


Si error: You have not concluded your merge (MERGE_HEAD exists).
faire alors un commit
git commit -m "..."

puis faire le pull 
git pull --rebase


Commandes séparées par ; 
la commande suivante s'exécute toujours

Commandes séparées par && 
la commande suivante ne s'exécute qu'en cas de succès de la commande précédente
